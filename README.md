# Документация REST API

## В чём заключалась задача
Разработать REST API для приёма и обработки данных о горных перевалах от мобильного приложения. API должен обеспечивать добавление новых записей, их просмотр, редактирование в определённых условиях и поиск по пользователю.

## Что было реализовано
1.  **POST /submitData**: Приём JSON-данных о новом перевале от мобильного приложения. Поля проходят валидацию, после чего запись сохраняется в базе данных PostgreSQL со статусом модерации "new".
2.  **GET /submitData/{id}**: Получение полной информации о существующем перевале по его уникальному идентификатору, включая текущий статус модерации.
3.  **PATCH /submitData/{id}**: Редактирование записи о перевале. Изменение разрешено только для записей со статусом "new". Данные пользователя (ФИО, email, телефон) при этом защищены от изменений.
4.  **GET /submitData/**: Получение списка всех перевалов, отправленных на сервер конкретным пользователем. Поиск осуществляется по email, переданному в query-параметре `user__email`.

## Как работать с API
Для взаимодействия с API необходимо отправлять HTTP-запросы к соответствующим эндпоинтам. Основные методы: `POST` для добавления данных, `GET` для их получения, `PATCH` для частичного обновления. Все данные передаются и получаются в формате JSON. Для работы требуется корректно настроенное подключение к базе данных PostgreSQL, параметры которого задаются через переменные окружения.

# Для запуска API необходимо:
1. Установить зависимости: pip install -r requirements.txt
2. Настроить параметры БД в файле .env
3. Запустить сервер: python run.py

Автоматическая документация доступна по адресу http://localhost:8000/docs
